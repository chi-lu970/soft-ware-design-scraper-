專案需求文件：Web 關鍵字爬蟲整合系統
1. 專案目標
開發一個簡易的 Web 應用程式，使用者可輸入關鍵字，系統將自動從目標網站爬取相關資訊，並以「視覺化表格」的形式即時顯示在網頁介面上。本專案重點在於軟體架構的整潔性與資料流的清晰度，以便後續產出需求分析報告與 UML 圖表。

2. 系統架構
本系統需遵循 MVC (Model-View-Controller) 或 三層架構 設計：

Frontend (View): 提供使用者介面。

Backend (Controller): 處理 API 請求與調度爬蟲。

Scraper Engine (Model): 執行實際的資料爬取與解析邏輯。

3. 技術棧要求 (Tech Stack)
語言: Python 3.x

前端框架: Streamlit (首選，因其開發快速且架構直觀) 或 Flask/FastAPI

爬蟲工具: Requests (處理 HTTP 請求), BeautifulSoup4 (解析 HTML)

資料處理: Pandas

4. 功能需求 (Functional Requirements)
搜尋介面:

提供一個輸入框讓使用者輸入 Keyword。

提供一個搜尋按鈕啟動爬蟲。

爬蟲邏輯 (核心):

目標: 預設爬取 [請自行填入目標網站，如：某新聞網] 的搜尋結果。

提取欄位:

標題 (Title): 網頁文章標題。

來源 (Source): 網域名稱或網站名稱。

關鍵句子 (Snippet): 在內文中搜尋關鍵字，並擷取其前後約 50 字的片段（非 AI 邏輯，單純字串匹配）。

原網址 (URL): 提供可跳轉的原始連結。

結果顯示:

以 Data Table (表格) 形式呈現結果。

URL 欄位必須是可點擊的連結，點擊後需另開新分頁。

5. 非功能性需求 (Non-functional Requirements)
模組化: 程式碼需物件導向化 (OOP)，定義明確的類別（例如 Scraper 類別），以便繪製 Class Diagram。

錯誤處理: 當網路連線失敗或找不到關鍵字時，需回傳友善的錯誤訊息。

爬蟲禮儀: 加入 time.sleep() 避免對目標伺服器造成負擔。

6. 實作建議 (給 AI 的實作指令)
「請依照上述需求，使用 Python 寫出一個結構化的爬蟲專案。請確保程式碼包含詳細註解，並將 UI 邏輯與 Scraper 邏輯分開存放。請優先使用 Streamlit 快速建立前端介面。不要使用任何深度學習或過於複雜的非同步技術，維持程式碼易讀性以利教學報告使用。」
